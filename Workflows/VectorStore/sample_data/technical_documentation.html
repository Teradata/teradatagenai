CloudSync Pro API Documentation

Overview

CloudSync Pro provides a comprehensive RESTful API for integrating cloud synchronization capabilities into your applications. The API supports multiple cloud providers and offers real-time synchronization, conflict resolution, and advanced security features.

Authentication

All API requests require authentication using API keys. Include your API key in the request header:

Authorization: Bearer YOUR_API_KEY

Obtaining API Keys

  1. Log in to your CloudSync Pro dashboard
  2. Navigate to Settings > API Keys
  3. Click "Generate New Key"
  4. Copy and securely store your API key

API Endpoints

1. Sync Files

POST /api/v1/sync

Initiates a synchronization operation between source and destination.

Parameters:

  • source_path (string, required): Path to source directory or file
  • destination_path (string, required): Path to destination
  • mode (string, optional): Sync mode - "incremental" or "full". Default: "incremental"
  • conflict_resolution (string, optional): "newest", "oldest", or "manual". Default: "newest"

2. Get Sync Status

GET /api/v1/sync/{sync_id}/status

Retrieves the current status of a synchronization job.

3. List Sync Jobs

GET /api/v1/sync/jobs

Returns a list of all synchronization jobs for the authenticated user.

Error Codes

  • 400 - Bad Request: Invalid parameters
  • 401 - Unauthorized: Invalid or missing API key
  • 403 - Forbidden: Insufficient permissions
  • 404 - Not Found: Resource does not exist
  • 429 - Too Many Requests: Rate limit exceeded
  • 500 - Internal Server Error: Contact support
